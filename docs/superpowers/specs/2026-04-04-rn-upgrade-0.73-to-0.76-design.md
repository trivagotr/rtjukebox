# React Native Upgrade: 0.73.0 → 0.76.9

**Date:** 2026-04-04
**Scope:** Mobile app only (`E:\rtmusicbox\mobile`)
**Goal:** Stay current, improve stability, lay groundwork for New Architecture

---

## Context

The app is currently on React Native 0.73.0. RN 0.76 is the milestone version where New Architecture (Fabric + TurboModules) became stable and enabled by default. Upgrading to 0.76 first (before going to 0.84) gives us a clean stepping stone.

---

## What Changes

### 1. package.json — npm dependencies

| Package | Current | Target |
|---|---|---|
| `react-native` | 0.73.0 | 0.76.9 |
| `react` | 18.2.0 | 18.3.2 |
| `react-test-renderer` | 18.2.0 | 18.3.2 |
| `@react-native/babel-preset` | ^0.73.18 | ^0.76.0 |
| `@react-native/eslint-config` | ^0.73.1 | ^0.76.0 |
| `@react-native/metro-config` | ^0.73.2 | ^0.76.0 |
| `@react-native/typescript-config` | ^0.73.1 | ^0.76.0 |
| `react-native-track-player` | ^4.0.1 | ^4.1.1 |

All other dependencies (navigation 6.x, zustand, axios, socket.io-client) are already compatible with RN 0.76 and stay unchanged.

### 2. Android — Gradle & build config

| File | Change |
|---|---|
| `android/build.gradle` | Kotlin 1.8.0 → 1.9.24, AGP 8.1.4 → 8.3.2 |
| `android/gradle/wrapper/gradle-wrapper.properties` | Gradle 8.3 → 8.6 |
| `android/app/build.gradle` | NDK 25.1.8937393 → 26.1.10909125 |
| `android/gradle.properties` | Add `newArchEnabled=false` |

New Architecture is explicitly disabled for this upgrade to keep scope contained. It can be enabled in a future upgrade.

### 3. iOS — Podfile

- Remove Flipper (deprecated since RN 0.74, removed in 0.75)
- Update `react_native_post_install` call signature for RN 0.76
- Update minimum iOS version if needed

### 4. Source code — 1 file

**`src/screens/RadioScreen.tsx`**
Remove `UIManager.setLayoutAnimationEnabledExperimental(true)` — this experimental flag is no longer needed in RN 0.76 as LayoutAnimation works out of the box on Android.

```tsx
// REMOVE this block:
if (Platform.OS === 'android') {
  if (UIManager.setLayoutAnimationEnabledExperimental) {
    UIManager.setLayoutAnimationEnabledExperimental(true);
  }
}
// Also remove UIManager from the react-native import
```

## What Does NOT Change

- All screen components (Radio, Podcast, Jukebox, Leaderboard, Profile, Auth)
- Navigation structure (RootNavigator)
- State management (Zustand, React Context)
- API/services layer
- Theme system
- TypeScript config (extends `@react-native/typescript-config`)
- Babel config (uses `@react-native/babel-preset`)
- Metro config (uses `@react-native/metro-config`)

## Upgrade Steps (Implementation Order)

1. **Update package.json** — bump all versions listed above
2. **Update android/build.gradle** — Kotlin + AGP versions
3. **Update gradle-wrapper.properties** — Gradle 8.6
4. **Update android/app/build.gradle** — NDK version
5. **Update android/gradle.properties** — add `newArchEnabled=false`
6. **Update ios/Podfile** — remove Flipper, update post-install
7. **Fix RadioScreen.tsx** — remove UIManager usage
8. **Clean install** — delete `node_modules`, run `npm install`
9. **Android build** — `cd android && ./gradlew clean`, then `npx react-native run-android`
10. **Verify** — app runs on device, all 4 tabs functional, audio playback works

---

## Risk & Rollback

- **Risk:** Low-medium. All key dependencies already support RN 0.76. The only source change is removing a deprecated flag.
- **Rollback:** `git checkout` restores everything — no DB or infra changes involved.
- **New Architecture:** Disabled via `newArchEnabled=false`. Zero risk from Fabric/TurboModules for this upgrade.
