# Requirements: RT Jukebox Verified Hardening

**Defined:** 2026-06-24
**Core Value:** Users can register or continue as guests, connect to the jukebox backend, discover songs, access games/gamification, and use the mobile/kiosk surfaces against a working backend without known dependency vulnerabilities.

## v1 Requirements

### Repository Setup

- [x] **SETUP-01**: Developer can clone the repository from `https://github.com/trivagotr/rtjukebox`.
- [x] **SETUP-02**: Dependencies install for backend, web controller, kiosk web, and mobile workspaces.

### Security

- [x] **SEC-01**: Backend dependency audit reports zero vulnerabilities.
- [x] **SEC-02**: Web controller dependency audit reports zero vulnerabilities.
- [x] **SEC-03**: Kiosk web dependency audit reports zero vulnerabilities.
- [x] **SEC-04**: Mobile dependency audit reports zero vulnerabilities.
- [x] **SEC-05**: Backend auth responses do not expose `password_hash`.
- [x] **SEC-06**: Production backend CORS configuration requires an explicit allowed-origin list.

### Backend and Database

- [x] **DB-01**: Database migration applies successfully against Docker Postgres.
- [x] **DB-02**: Registered user creation succeeds against the real database and returns an access token.
- [x] **DB-03**: Authenticated profile lookup succeeds for a registered user.
- [x] **DB-04**: Guest user creation persists `role = 'guest'` and `is_guest = true`.
- [x] **DB-05**: Schema contains `last_ip`, `user_agent`, and `last_super_vote_at` columns used by runtime routes.

### Jukebox and Mobile Contract

- [x] **JBOX-01**: Mobile jukebox search uses structured Axios query params.
- [x] **JBOX-02**: Backend `GET /api/v1/jukebox/songs` returns HTTP 200 and a `data.items` array for mobile consumption.
- [x] **JBOX-03**: Authenticated gamification games endpoint returns HTTP 200.

### Mobile Readiness

- [x] **MOB-01**: Mobile lint exits successfully.
- [x] **MOB-02**: Mobile test suite passes.
- [x] **MOB-03**: React Native config resolves.
- [x] **MOB-04**: Android mobile debug APK builds.
- [x] **MOB-05**: Android automotive debug APK builds.

### Web and Kiosk

- [x] **WEB-01**: Web controller lint, tests, and production build pass.
- [x] **KIOSK-01**: Kiosk web tests pass on the upgraded dependency set.

## v2 Requirements

### Release Operations

- **REL-01**: Production deployment environment is provisioned and smoke-tested.
- **REL-02**: Android release signing and distribution artifacts are configured.
- **REL-03**: Physical-device mobile and automotive QA is completed.

| Feature | Reason |

| Store release signing | Debug APK readiness was enough for this milestone. |
| New gameplay design | Existing games/gamification readiness was verified through backend and mobile tests. |
